# Phase 5: Dashboard (v0)

Static site rendering the v0 sentiment-rate barometer.

## State

Prototype with **mock data**. Real data lands when the v0 classifier
(`scripts/v0_classify.py`, pending) writes `phase5/data.json` from the corpus.

## Layout

```
phase5/
├── index.html          single-page dashboard + methodology
├── style.css           palette is in :root CSS variables
├── dashboard.js        renders SVG charts from data.json (currently mock_data.json)
├── mock_data.json      placeholder — same schema the classifier will produce
├── README.md           this file
└── (future: data.json) generated by Phase 3, gitignored
```

## What the dashboard shows

Per model, three things plus a methodology section:

1. **Big number**: % of mentions this week classified as complaints, with
   delta from last week.
2. **Trend line**: 90 days, weekly. Release events appear as dashed verticals.
3. **Top 10 terms** in complaint subset (word/bigram frequency, no clustering).
4. **Defection rhetoric line**: rate of "switching to / cancelled / done with"
   phrases. Labeled as rhetoric, not behavior.

## Locked design decisions

- Single-page dashboard. Methodology section is on the same page (anchor link).
- Sticky banner discloses the two key caveats (selection bias, classifier
  undercount) before any chart is read.
- X-axis toggles between wall-clock time and weeks-since-release.
- Defection language is labeled as **rhetoric**, never as user behavior.
- Side-by-side claude / openai panels with identical y-axis treatment.
- **No fine-grained category time series.** That was the v1 plan; archived.
  Top-N word frequency is the only texture v0 shows.

## Aesthetic

Dark monospace, lemon-yellow accents. Palette in `:root` CSS custom properties
(`--lemon-rind`, `--lemon-flesh`, `--lemon-pith`, `--lemon-leaf`,
`--lemon-zest`). Swap in real palette there.
