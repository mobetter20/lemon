# Phase 5: Dashboard

Static site rendering the counter-benchmark. Single page, no frameworks,
vanilla HTML/CSS/JS with SVG charts.

## State

Prototype with **mock data**. Real data lands when Phase 3 (classifier)
produces `phase5/data.json` from the corpus.

## Layout

```
phase5/
├── index.html          single-page dashboard + methodology
├── style.css           palette is in :root CSS variables — swap in real palette here
├── dashboard.js        renders SVG charts from data.json (currently mock_data.json)
├── mock_data.json      placeholder — same schema Phase 3 will produce
├── README.md           this file
└── (future: data.json) generated by Phase 3 → committed to data branch
```

## Aesthetic

Dark monospace, lemon-yellow accents, slight roundness. Palette is in
`:root` CSS custom properties (`--lemon-rind`, `--lemon-flesh`, etc.).
Swap real palette in there once defined.

## Locked design decisions

- Single-page dashboard. Methodology section is on the same page (anchor link).
- Sticky methodology disclosure banner above the chart — co-equal UI surface,
  not a footer link.
- X-axis toggles between wall-clock time and weeks-since-release per model.
- Defection/loyalty markers labeled as **rhetoric**, never as user behavior.
- Side-by-side claude / openai panels with identical y-axes for honest visual
  comparison.
